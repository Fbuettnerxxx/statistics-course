Concept

1. Confidence Intervals for Means

A confidence interval gives a range of plausible values for a population parameter based on sample data.

Confidence Interval for a Mean
x̄ ± t*df · SE     where     SE = s / √n     and     df = n − 1

The margin of error is the ± part: ME = t*df · SE. You can also compute it as: ME = upper bound − x̄ (or x̄ − lower bound).

The #1 misinterpretation (exam trap!):
WRONG: "There is a 95% probability that the true mean is in this interval."
RIGHT: "We are 95% confident that this interval captures the true mean." This means: if we repeated the study many times, 95% of the resulting intervals would contain the true mean. It's about the method's long-run success rate, not a probability statement about one specific interval.
Width of CI depends on:
• Higher confidence level (e.g. 99% vs 95%) → wider CI (bigger t*)
• Larger sample size n → narrower CI (smaller SE)
• Larger standard deviation s → wider CI (larger SE)
Quick example:
n = 25, x̄ = 68, s = 10. Find 95% CI.
SE = 10 / √25 = 2.0
df = 24, t*24,0.025 ≈ 2.064
CI = 68 ± 2.064 × 2.0 = 68 ± 4.128 = (63.87, 72.13)
Margin of error = 4.128
Concept

2. Confidence Intervals for Proportions

When the parameter of interest is a proportion (e.g. "what fraction of voters support X?"), use the z-distribution instead of t.

CI for a Single Proportion
p̂ ± z* · SE     where     SE = √(p̂(1 − p̂) / n)
Success-failure condition (must check!):
np̂ ≥ 10    AND    n(1 − p̂) ≥ 10
If this fails, the normal approximation isn't valid and you should use exact binomial methods.

Difference in Two Proportions

CI for Difference in Proportions
(p̂1 − p̂2) ± z* · SE

SE = √(p̂1(1 − p̂1)/n1 + p̂2(1 − p̂2)/n2)
Common z* values (memorize these):
90% CI → z* = 1.645
95% CI → z* = 1.960
99% CI → z* = 2.576
Concept

3. Bootstrap Confidence Intervals

Bootstrapping is a simulation-based approach to constructing confidence intervals. Instead of relying on formulas and distributional assumptions (normality, known form of SE), you let the data itself tell you about sampling variability.

What is Bootstrapping?

  1. Start with your original sample of size n.
  2. Resample with replacement from the original sample to create a new sample of the same size n.
  3. Compute the statistic (e.g. mean, proportion) on the resampled data.
  4. Repeat many times (typically 1,000–15,000 times) to build a bootstrap distribution.
Key idea: The bootstrap distribution approximates the sampling distribution of the statistic. The spread of the bootstrap distribution estimates the standard error.

Method 1: Bootstrap SE Method

Bootstrap SE Confidence Interval
x̄ ± z* · SEbootstrap

where SEbootstrap = standard deviation of the bootstrap distribution (i.e. the SD of all the resampled statistics).

Method 2: Percentile Method

Use the quantiles of the bootstrap distribution directly as CI bounds:

Percentile Bootstrap CI (95%)
(2.5th percentile of bootstrap stats,   97.5th percentile of bootstrap stats)
When to use which method?
• SE method: works well when the bootstrap distribution is approximately symmetric/normal.
• Percentile method: more robust, works even when the bootstrap distribution is skewed. This is the default in most modern statistics courses.

The infer Workflow for Bootstrap CI

The infer package in R provides a tidy, pipeable workflow for bootstrap inference. This is the workflow tested on TUM exams:

# Step 1: Generate the bootstrap distribution boot_dist <- data |> specify(response = var) |> generate(reps = 1000, type = "bootstrap") |> calculate(stat = "mean") # Step 2: Get the CI (percentile method) boot_dist |> get_ci(level = 0.95, type = "percentile") # Or: SE method boot_dist |> get_ci(level = 0.95, type = "se", point_estimate = mean(data$var))
The 4 verbs of infer (exam essential):
1. specify() — declare the response variable (and explanatory variable if applicable)
2. hypothesize() — state the null hypothesis (only for hypothesis tests, NOT for bootstrap CI)
3. generate() — create resamples (type = "bootstrap" for CI, "draw" or "permute" for tests)
4. calculate() — compute the statistic on each resample

For Proportions

When bootstrapping a proportion, specify the variable as a factor and declare which level counts as "success":

# Bootstrap CI for a proportion boot_dist <- data |> specify(response = outcome, success = "yes") |> generate(reps = 1000, type = "bootstrap") |> calculate(stat = "prop")
Example: Emergency expense survey
A survey of 840 adults finds that 268 cannot cover a $400 emergency expense.

Observed proportion: p̂ = 268 / 840 = 0.319

Bootstrap approach:
1. Resample 840 adults (with replacement) from the original 840 responses.
2. Compute the proportion who cannot cover the expense in each resample.
3. Repeat 1,000 times → get 1,000 bootstrap proportions.
4. The middle 95% of these proportions forms the 95% CI.

Suppose the bootstrap gives: SEboot = 0.0161
SE method: 0.319 ± 1.96 × 0.0161 = (0.288, 0.351)

Compare to the formula-based CI:
SE = √(0.319 × 0.681 / 840) = √(0.000259) = 0.0161
CI = 0.319 ± 1.96 × 0.0161 = (0.288, 0.351)

In this case, the bootstrap and formula results are nearly identical because the sample is large and the distribution is symmetric. The bootstrap shines when conditions for the formula are not met.
Exam trap: Do NOT confuse generate(type = "bootstrap") (used for confidence intervals) with generate(type = "draw") or generate(type = "permute") (used for hypothesis tests). The exam will ask you to fill in the blanks and choosing the wrong type is a common mistake.
Concept

4. Hypothesis Testing Framework

Hypothesis testing is a formal procedure to decide whether data provides enough evidence against a null hypothesis.

H₀ (Null Hypothesis) The "nothing interesting" claim.
Always contains = sign.
Example: H₀: μ = 20
HA (Alternative Hypothesis) What we're trying to show.
Contains ≠, <, or >.
Example: HA: μ ≠ 20

One-sided vs Two-sided

  • Two-sided: HA: μ ≠ μ₀ → "different from" (most common)
  • One-sided (right): HA: μ > μ₀ → "greater than"
  • One-sided (left): HA: μ < μ₀ → "less than"

The 4-Step Procedure

  1. State hypotheses: H₀ and HA
  2. Compute test statistic: T or Z (how many SEs from H₀)
  3. Find p-value: probability of getting a result this extreme if H₀ were true
  4. Decision: if p-value < α → reject H₀. If p-value ≥ α → fail to reject H₀.
NEVER say "accept H₀"!
The correct phrasing is "fail to reject H₀." We don't prove H₀ is true — we just didn't find enough evidence against it.

Type I and Type II Errors

H₀ is actually trueH₀ is actually false
Reject H₀Type I Error (α)Correct! (Power = 1−β)
Fail to reject H₀Correct!Type II Error (β)
α = P(reject H₀ | H₀ true) = significance level (usually 0.05)
β = P(fail to reject H₀ | H₀ false) = missed discovery rate
Power = 1 − β = P(reject H₀ | H₀ false) = ability to detect a real effect
Concept

5. t-Test for Means

One-Sample t-Test

Test Statistic
T = (x̄ − μ₀) / SE     where     SE = s / √n     df = n − 1

Two-Sample t-Test (Independent Groups)

Test Statistic
T = (x̄1 − x̄2) / SE     where     SE = √(s1²/n1 + s2²/n2)

Paired t-Test

When data comes in pairs (before/after, matched subjects), compute the differences di = x1i − x2i, then do a one-sample t-test on the differences.

Paired Test Statistic
T = d̄ / (sd / √n)     where d̄ = mean of differences, sd = SD of differences

Reading R Output (Exam Essential)

Typical R output you'll see:
t = 2.45, df = 28, p-value = 0.021 alternative hypothesis: true mean is not equal to 50 95 percent confidence interval: 51.23 58.77 sample estimates: mean of x 55.00
Read it as: T = 2.45, 28 degrees of freedom, two-sided p = 0.021.
Since p = 0.021 < 0.05 → reject H₀: μ = 50 at α = 0.05.
Exam shortcut — CI-test connection:
If μ₀ is NOT inside the CI → reject H₀
If μ₀ is inside the CI → fail to reject H₀
(This works for two-sided tests at the corresponding α level)
p-value for two-sided vs one-sided:
R's t.test() gives two-sided p-value by default.
For a one-sided test: pone-sided = ptwo-sided / 2
So pt(t, df) gives the one-sided (left-tail) probability. For a two-sided p-value: 2 × pt(−|t|, df).
Concept

6. Proportion Tests (Z-test)

One-Proportion Z-Test

Test Statistic
Z = (p̂ − p₀) / SE₀     where     SE₀ = √(p₀(1 − p₀) / n)
SE for proportions — test vs CI (exam classic!):
• For a hypothesis test: use the null value p₀ in the SE formula: SE = √(p₀(1 − p₀) / n)
• For a confidence interval: use the observed value p̂ in the SE formula: SE = √(p̂(1 − p̂) / n)
Mixing these up is one of the most common exam errors.

Two-Proportion Z-Test

Pooled Proportion & Test Statistic
p̂pool = (x1 + x2) / (n1 + n2)

Z = (p̂1 − p̂2) / √(p̂pool(1 − p̂pool)(1/n1 + 1/n2))
Why pooled proportion?
Under H₀: p1 = p2, we assume the two groups have the same population proportion. The pooled proportion p̂pool is the best estimate of that common proportion. Note: x1 and x2 are the counts of successes (not proportions), so p̂pool = total successes / total observations.

Exact Binomial Test (for small samples)

When the normal approximation doesn't hold (np₀ < 10 or n(1−p₀) < 10), use the exact binomial.

Using pbinom in R:
H₀: p = 0.5, HA: p > 0.5. Observe 8 successes in 10 trials.
p-value = P(X ≥ 8 | p = 0.5) = 1 − P(X ≤ 7) = 1 − pbinom(7, 10, 0.5)

Key: For HA: p > p₀ → p-value = 1 − pbinom(x − 1, n, p₀) = P(X ≥ x)
For HA: p < p₀ → p-value = pbinom(x, n, p₀) = P(X ≤ x)
Exam watch: The TUM exam frequently asks you to compute p-values using pbinom for small-sample proportion tests. Make sure you know whether to use pbinom(x, n, p) or 1 − pbinom(x−1, n, p) depending on the direction of HA.
Concept

7. Simulation-Based Hypothesis Testing

Instead of using formulas (Z or T statistics) to compute p-values, simulation-based testing generates a null distribution by simulating data under the null hypothesis, then checks how extreme the observed statistic is.

Core Idea

  1. Assume H₀ is true and simulate many datasets under that assumption.
  2. Compute the test statistic for each simulated dataset → this gives the null distribution.
  3. Compare the observed statistic to the null distribution.
  4. p-value = proportion of simulated statistics that are as extreme as or more extreme than the observed statistic.
Simulation-Based p-value
p-value = (number of simulated stats as extreme as observed) / (total reps)

Testing a Single Proportion (type = "draw")

To test H₀: p = p₀, simulate draws from a population where the true proportion is p₀:

# Test H₀: p = 0.10 against Hₐ: p ≠ 0.10 # Observed: 15 out of 100 (p̂ = 0.15) null_dist <- data |> specify(response = outcome, success = "yes") |> hypothesize(null = "point", p = 0.10) |> generate(reps = 1000, type = "draw") |> calculate(stat = "prop") # Get p-value null_dist |> get_p_value(obs_stat = 0.15, direction = "two-sided")
hypothesize(null = "point", p = ...) — used when testing a single proportion against a specific value. The type = "draw" in generate() means: draw new samples of size n from a population where p = p₀.

Testing Independence / Comparing Groups (type = "permute")

To test whether two groups are different (H₀: no association), permute the group labels to break any real association:

# Test H₀: p₁ = p₂ (independence) # e.g., Does treatment affect outcome? null_dist <- data |> specify(outcome ~ group, success = "yes") |> hypothesize(null = "independence") |> generate(reps = 1000, type = "permute") |> calculate(stat = "diff in props", order = c("treatment", "control"))
hypothesize(null = "independence") — used when comparing two groups. The type = "permute" means: shuffle the group labels randomly. If H₀ is true (groups are the same), shuffling should not change the statistic much.

Summary: Which generate() type to use?

Goalgenerate(type = ?)hypothesize(null = ?)
Bootstrap CI"bootstrap"not used
Test single proportion/mean"draw""point"
Test independence (2 groups)"permute""independence"
Simulation vs Formula — when does it matter?
• For large samples, simulation-based and formula-based p-values are very close.
• Simulation is preferred when sample sizes are small, distributions are skewed, or the test statistic doesn't have a known theoretical distribution.
• On the exam, you'll typically be asked to fill in the infer pipeline or interpret simulation output, not run actual simulations.
Interpreting simulation output:
If you simulate 1,000 datasets under H₀ and find that 38 of them produce a test statistic as extreme as your observed statistic:

p-value = 38 / 1000 = 0.038

At α = 0.05: p-value = 0.038 < 0.05 → reject H₀.
Concept

8. Power & Sample Size

Power = probability of correctly rejecting H₀ when it's false = 1 − β. Higher power = better ability to detect a real effect.

Sample Size Formula (Two-Sample Test for Means)

Minimum Sample Size per Group
n ≥ (2σ₀² / δ₀²) · (z1−α/2 + z1−β)²

Where:

  • σ₀ = assumed standard deviation
  • δ₀ = minimum effect size (difference) you want to detect
  • z1−α/2 = z-value for significance level (e.g. 1.96 for α = 0.05)
  • z1−β = z-value for power (e.g. 0.524 for 70% power, 0.842 for 80% power)
Always round UP!
If the formula gives n = 23.1, you need n = 24 per group. You can't have a fraction of a person.

Reading power.t.test() in R

R output example:
power.t.test(delta = 2, sd = 5, sig.level = 0.05, power = 0.80, type = "two.sample") n = 100.0538
This means you need 101 per group (round up!) = 202 total.
What increases power?
• Larger sample size n ↑
• Larger effect size δ₀ ↑
• Larger α (less strict) ↑
• Smaller variability σ ↓
Cheat Sheet

9. CI & Test Connection + Summary

The CI-Test Duality

A (1−α) CI corresponds to a two-sided test at significance level α:
• 90% CI ↔ α = 0.10 test
• 95% CI ↔ α = 0.05 test
• 99% CI ↔ α = 0.01 test

If the hypothesized value falls outside the CI → reject H₀
If the hypothesized value falls inside the CI → fail to reject H₀

Module 4 Master Summary

  1. CI for mean: x̄ ± t* · (s/√n), df = n−1
  2. CI for proportion: p̂ ± z* · √(p̂(1−p̂)/n), check np̂ ≥ 10
  3. Bootstrap CI: resample with replacement → SE method (x̄ ± z* · SEboot) or percentile method (use quantiles directly)
  4. Hypothesis test: H₀ → test stat → p-value → compare α → conclude
  5. t-test: T = (estimate − null) / SE. Paired = test differences.
  6. Proportion Z-test: use p₀ in SE for test, p̂ in SE for CI
  7. Pooled proportion: p̂pool = (x1 + x2) / (n1 + n2) — use in two-proportion test SE
  8. Small samples: use pbinom for exact binomial test
  9. Simulation-based test: generate null distribution via infer, p-value = extreme simulations / total reps
  10. infer types: "bootstrap" for CI, "draw" for point null, "permute" for independence
  11. Power: n ≥ (2σ²/δ²)(z1−α/2 + z1−β)², always round UP
  12. Never say "accept H₀" — always "fail to reject H₀"
  13. CI ↔ test: value outside CI = reject, value inside CI = fail to reject

Exercise 1 — t-test R Output True/False
4 pts • Appears every exam
The following R output is given:
t = 0.2024, df = 14, p-value = 0.8425 alternative hypothesis: true mean is not equal to 20 95 percent confidence interval: 18.91483 21.31131 sample estimates: mean of x 20.11307
Determine whether each statement is true or false. Justify your answer. a) The null hypothesis tested here is H₀: μ = 20.11307. b) pt(0.2024, df=14) will result in 0.8425. c) The margin of error is 1.19824. d) [18.97138, 21.25476] is a 99% CI for the true mean.
a)
b)
c)
d)
a) FALSE.
The null hypothesis is H₀: μ = 20, NOT μ = 20.11307. The R output says "true mean is not equal to 20" which means the alternative is HA: μ ≠ 20, so the null is H₀: μ = 20. The value 20.11307 is the sample mean x̄, not the hypothesized value.

b) FALSE.
pt(0.2024, df=14) gives the one-sided (left-tail) cumulative probability P(T ≤ 0.2024). The p-value 0.8425 is the two-sided p-value. For a two-sided test: p-value = 2 × (1 − pt(|t|, df)) = 2 × (1 − pt(0.2024, 14)). So pt(0.2024, 14) ≈ 0.5788, not 0.8425.
Alternatively: 2 × pt(−0.2024, 14) = 2 × (1 − 0.5788) = 2 × 0.4212 = 0.8425.

c) TRUE.
Margin of error = upper bound − x̄ = 21.31131 − 20.11307 = 1.19824.
Check: x̄ − lower bound = 20.11307 − 18.91483 = 1.19824. Consistent.

d) FALSE.
A 99% CI is wider than a 95% CI (higher confidence requires a wider interval). The interval [18.97138, 21.25476] is narrower than [18.91483, 21.31131], so it cannot be a 99% CI. It could potentially be a 90% CI (narrower = less confidence), but definitely not 99%.

Exercise 2 — Paired t-test CI
2 pts • 2019 Exam
A paired t-test yields p-value = 0.03 for H₀: μd = 0 vs HA: μd ≠ 0. Which of these is the corresponding 95% confidence interval? Option A: (1.3, 3.4) Option B: (−1.3, 3.4) Justify your choice.
Answer: Option A — (1.3, 3.4)

Reasoning using the CI-test connection:
The p-value is 0.03 < 0.05, so we reject H₀: μd = 0 at α = 0.05.

This means the hypothesized value μd = 0 must be outside the 95% CI.

• Option A: (1.3, 3.4) — 0 is NOT in this interval. Consistent with rejecting H₀.
• Option B: (−1.3, 3.4) — 0 IS in this interval. This would mean we fail to reject H₀, which contradicts p = 0.03.

Therefore, the correct CI is (1.3, 3.4).

Exercise 4 — Proportion Test with Binomial
4 pts • 2019/2023 Exam
1.7% of the US population lives on less than $2 per day. A sample of 100 people in Atlanta found that 4% live on less than $2 per day. a) Set up the hypotheses to test whether the proportion in Atlanta is significantly higher than the national rate. Use α = 0.05. b) Can we use the normal approximation? Check the conditions. c) Compute the p-value using the exact binomial test and state your conclusion.
Hint: pbinom(3, 100, 0.017) = 0.9367
a)
b)
a) Hypotheses:
H₀: p = 0.017 (Atlanta has the same rate as the national average)
HA: p > 0.017 (Atlanta has a higher rate) — one-sided test

b) Check normal approximation conditions:
np₀ = 100 × 0.017 = 1.7 < 10 ✗
n(1 − p₀) = 100 × 0.983 = 98.3 ≥ 10 ✓

Since np₀ = 1.7 < 10, the success-failure condition is NOT met. We cannot use the normal approximation and must use the exact binomial test.

c) Exact binomial p-value:
We observed x = 4 successes (4% of 100).
HA: p > 0.017 (right-sided), so:

p-value = P(X ≥ 4 | n=100, p=0.017)
= 1 − P(X ≤ 3)
= 1 − pbinom(3, 100, 0.017)
= 1 − 0.9367
= 0.0633

Conclusion: Since p-value = 0.0633 > α = 0.05, we fail to reject H₀. There is not sufficient evidence at the 5% significance level to conclude that the proportion of people in Atlanta living on less than $2/day is significantly higher than the national rate of 1.7%.

Exercise 5 — Margin of Error for Difference in Proportions
3 pts • 2019/2023 Retake
In a clinical study, 16 out of 20 patients in the treatment group improved, and 45 out of 78 patients in the control group improved. Compute the margin of error for a 95% confidence interval for the difference in proportions (ptreatment − pcontrol).
Step 1: Compute sample proportions
p̂1 = 16/20 = 0.80 (treatment)
p̂2 = 45/78 = 0.5769 (control)

Step 2: Compute SE for the difference
SE = √(p̂1(1−p̂1)/n1 + p̂2(1−p̂2)/n2)
SE = √(0.80 × 0.20 / 20 + 0.5769 × 0.4231 / 78)
SE = √(0.16/20 + 0.2441/78)
SE = √(0.008 + 0.003129)
SE = √0.011129
SE = 0.10549

Step 3: Margin of error
ME = z* × SE = 1.96 × 0.10549 = 0.2068

The margin of error is approximately 0.207 (or about 20.7 percentage points).

Note: The point estimate is p̂1 − p̂2 = 0.80 − 0.5769 = 0.2231, so the 95% CI would be (0.2231 − 0.207, 0.2231 + 0.207) = (0.016, 0.430). Since 0 is not in the CI, the difference is statistically significant.

Exercise 6 — Power & Sample Size
3 pts • 2023 Exam
A university surveys students about course satisfaction. The average number of completed surveys is 4 per student with a standard deviation of 2.2. The university wants to detect an effect size of δ₀ = 1 additional survey (i.e., increase from 4 to 5). Using α = 0.05 and power = 70%, how many students are needed per group (two-sample test)?
Hint: qnorm(0.975) = 1.9600, qnorm(0.70) = 0.5244
Given:
σ₀ = 2.2, δ₀ = 1, α = 0.05, power = 0.70 (so β = 0.30)

Step 1: Find z-values
z1−α/2 = z0.975 = qnorm(0.975) = 1.9600
z1−β = z0.70 = qnorm(0.70) = 0.5244

Step 2: Apply formula
n ≥ (2σ₀² / δ₀²) · (z1−α/2 + z1−β)²

n ≥ (2 × 2.2² / 1²) · (1.9600 + 0.5244)²
n ≥ (2 × 4.84 / 1) · (2.4844)²
n ≥ 9.68 × 6.1722
n ≥ 59.75

Step 3: Round UP
n = 60 per group (120 total)

The university needs at least 60 students per group to detect an increase of 1 survey with 70% power at the 5% significance level.

Exercise 7 — Roulette Fairness (Simulation-Based Test)
5 pts • 2023 Retake Exam
A casino patron suspects a roulette wheel is unfair. On a standard European roulette wheel, the probability of landing on red is P(red) = 18/37 ≈ 0.4865. The patron plays 500 games and red lands 260 times (p̂ = 260/500 = 0.52). We want to test whether the wheel is fair using a simulation-based approach with the infer package. a) State the hypotheses. b) Fill in the blanks in the following infer pipeline:
null_dist <- roulette_data |> specify(response = _______, success = _______) |> hypothesize(null = _______, p = _______) |> generate(reps = 1000, type = _______) |> calculate(stat = _______)
c) The simulation produces the following result: out of 1,000 simulated datasets under H₀, 112 had a proportion of red ≥ 260/500 = 0.52. Compute the p-value and state your conclusion at α = 0.05. d) Would your conclusion change if we used a two-sided test? Explain.
a) Hypotheses
b) Fill in the blanks
c) p-value and conclusion
d) Two-sided test?
a) Hypotheses:
H₀: p = 18/37 ≈ 0.4865 (the wheel is fair)
HA: p > 18/37 (red lands more often than expected — one-sided test)
Or, if testing for any deviation: HA: p ≠ 18/37 (two-sided)

b) Completed pipeline:
null_dist <- roulette_data |> specify(response = color, success = "red") |> hypothesize(null = "point", p = 18/37) |> generate(reps = 1000, type = "draw") |> calculate(stat = "prop")
Explanation of each blank:
• response = color — the variable recording the outcome of each spin (e.g., "red" or "not red")
• success = "red" — which level counts as a "success"
• null = "point" — we're testing against a specific value (not independence)
• p = 18/37 — the null hypothesis proportion
• type = "draw" — simulate new samples by drawing from a population where p = 18/37
• stat = "prop" — compute the proportion of reds in each simulated sample

c) p-value (one-sided):
Out of 1,000 simulations, 112 had a proportion ≥ 0.52.

p-value = 112 / 1000 = 0.112

Conclusion: Since p-value = 0.112 > α = 0.05, we fail to reject H₀. There is not sufficient evidence to conclude that the roulette wheel is unfair. The observed proportion of 0.52 is within the range of what we would expect from random chance with a fair wheel.

d) Two-sided test:
For a two-sided test, we would also count simulations with proportions as extreme in the other direction (i.e., ≤ 2×0.4865 − 0.52 = 0.453).
Approximately: p-valuetwo-sided ≈ 2 × 0.112 = 0.224.
This is even larger than 0.05, so the conclusion would not change — we would still fail to reject H₀. If anything, the two-sided test is more conservative (harder to reject).
