1. K-Nearest Neighbors (KNN)

KNN is a non-parametric classification method. It does not learn coefficients or fit a model equation. Instead, it classifies a new observation by looking at the k closest training points and taking a majority vote.

How KNN works:

  1. Choose a value for k (number of neighbors).
  2. For a new observation x0, compute the distance from x0 to every training point.
  3. Find the k nearest training points (smallest distances).
  4. Classify x0 by majority vote among those k neighbors.
Euclidean Distance (the standard distance measure)
d(x, x') = √((x₁ − x₁')² + (x₂ − x₂')² + ... + (xp − xp')²)

In words: square the difference for each predictor, sum them up, take the square root.

The role of k:

Small k (e.g., k = 1) Very flexible decision boundary
Low bias, high variance
Overfits — captures noise
Sensitive to individual training points
Large k (e.g., k = n) Very rigid decision boundary
High bias, low variance
Underfits — too simple
With k = n, always predicts the majority class
Exam tip: KNN is the opposite of linear regression for flexibility. In linear regression, more predictors = more flexible. In KNN, smaller k = more flexible. The flexibility parameter goes in the opposite direction!

2. The Bias-Variance Tradeoff

This is one of the most fundamental concepts in statistical learning. Every prediction error can be decomposed into three components:

Bias-Variance Decomposition
E[(Y − f̂(x))²] = Var(f̂(x)) + [Bias(f̂(x))]² + Var(ε)

What each term means:

  • Bias = E[f̂(x)] − f(x) — How far off is the model on average from the true relationship? A model that is too simple (e.g., fitting a line to a curve) has high bias because it systematically misses the true pattern.
  • Variance = Var(f̂(x)) — How much does the model change when trained on different data sets? A model that is too complex (e.g., fitting a 20th-degree polynomial) has high variance because it changes wildly with each new training set.
  • Irreducible error = Var(ε) — The noise in the data that no model can ever eliminate. This is the floor of prediction error.
The tradeoff: As model complexity increases, bias decreases but variance increases. The goal is to find the sweet spot that minimizes total test error (bias² + variance + irreducible error).

Visual intuition:

Think of a dartboard:
  • Low bias, low variance: Darts clustered tightly around the bullseye. (Best case!)
  • Low bias, high variance: Darts scattered, but centered on the bullseye. (Overfitting)
  • High bias, low variance: Darts clustered tightly, but away from the bullseye. (Underfitting)
  • High bias, high variance: Darts scattered and off-center. (Worst case!)

Parametric vs. Non-Parametric Methods

Parametric Assumes a specific form for f(x)
e.g., linear regression, logistic regression

Pros: Easier to estimate, needs less data
Cons: If the assumed form is wrong → high bias
Non-Parametric Makes NO assumption about the form of f(x)
e.g., KNN, decision trees, splines

Pros: Can fit any shape → low bias
Cons: Needs much more data, risk of overfitting (high variance)
When to prefer non-parametric methods:
• When you have a large number of training observations
• When the true relationship is likely non-linear or complex
• When you care more about prediction accuracy than interpretability

When to prefer parametric methods:
• When you have a small number of training observations
• When the true relationship is approximately linear
• When you need interpretable coefficients (e.g., odds ratios)
Exam warning: The exam loves to ask "In what situation would a non-parametric method outperform a parametric method?" Answer: When the true f(x) is highly non-linear and there is enough training data. A parametric method would have high bias because it forces a linear (or other assumed) shape on a non-linear reality.

3. Relative Risk vs. Odds Ratio

Before diving into logistic regression, it is important to understand two measures of association: relative risk and odds ratio.

Relative Risk (RR)
RR = P(disease | exposed) / P(disease | unexposed)

Relative risk compares the probability (risk) of an outcome in two groups directly.

Example:
DiseaseNo diseaseTotal
Exposed3070100
Unexposed1090100
P(disease | exposed) = 30/100 = 0.30
P(disease | unexposed) = 10/100 = 0.10

RR = 0.30 / 0.10 = 3.0
"Exposed individuals are 3 times as likely to develop the disease as unexposed individuals."

How RR differs from Odds Ratio:

Relative Risk (RR) Ratio of probabilities
RR = P(D|E) / P(D|~E)

More intuitive interpretation
Used in prospective studies (cohort studies)
Odds Ratio (OR) Ratio of odds
OR = [P(D|E)/(1−P(D|E))] / [P(D|~E)/(1−P(D|~E))]

What logistic regression gives us
Used in case-control and cross-sectional studies
Key relationship: When the disease/outcome is rare (prevalence < 10%), OR ≈ RR. This is called the rare disease assumption. When the outcome is common, the OR exaggerates the effect compared to RR.

Using the example above:
Odds(disease | exposed) = 30/70 = 0.4286
Odds(disease | unexposed) = 10/90 = 0.1111
OR = 0.4286 / 0.1111 = 3.86 (larger than RR = 3.0 because disease is not rare)

4. Logistic Regression Basics

When the response variable is binary (0/1, yes/no, survived/died), we can't use linear regression — predictions would go below 0 or above 1. Instead, we model the log-odds.

The Logistic Regression Model
logit(p) = log(p / (1 − p)) = β₀ + β₁x₁ + β₂x₂ + ...

The key insight: we do not model the probability directly. We model the log-odds (also called the logit). The logit can range from −∞ to +∞, so a linear model makes sense.

R Code — Fitting a Logistic Regression
model <- glm(y ~ x1 + x2, data = train, family = binomial)
Exam essentials:
• glm() instead of lm()
• family = binomial is the magic argument — this tells R it's logistic
• Coefficients are in log-odds, NOT probabilities
• To get odds ratios: exponentiate the coefficients with eβ
Where logistic regression fits in the bias-variance framework: Logistic regression is a parametric method. It assumes the log-odds are a linear function of the predictors. This means it has relatively high bias (if the true relationship is non-linear) but low variance (stable across different training sets). Compared to KNN, logistic regression is less flexible but more interpretable.
Common exam mistake: Writing family = "binomial" with quotes. In R, binomial is a function, not a string. Both work in practice, but the exam expects family = binomial without quotes.

5. Odds Ratios — Interpreting Coefficients

In logistic regression, each coefficient β is a log-odds ratio. To get the odds ratio (OR), exponentiate:

Odds Ratio
OR = eβ

Interpretation for continuous variables:

"For each one-unit increase in x, the odds of success are multiplied by eβ."

Example: β = 0.85 → OR = e0.85 = 2.34
"For each one-unit increase in x, the odds of success are 2.34 times as large."

Interpretation for categorical variables:

"The odds for group A are eβ times the odds for the reference group."

Example: βSexmale = −2.71 → OR = e−2.71 = 0.067
"The odds of survival for males are 0.067 times the odds for females" — meaning males have much lower odds.

Interpretation for interaction terms:

With interactions, compute the combined slope first!
If the model has Pclass + Sexmale + Pclass:Sexmale, then:
• Slope of Pclass for females: βPclass
• Slope of Pclass for males: βPclass + βPclass:Sexmale
Then compute OR = ecombined slope for that specific group.
Exam rule: You MUST describe the odds ratio in words. Writing "OR = 2.34" alone gets zero points. Always say what changes, by how much, and in which direction.

6. Prediction & Classification

After fitting the model, we want to predict the probability of success and classify observations.

R Code — Getting Predicted Probabilities
pred_probs <- predict(model, newdata = test, type = "response")
type = "response" is crucial!
• type = "response" → returns probabilities (between 0 and 1)
• Without it (default) → returns log-odds (can be any number)
The exam will always want probabilities.

From probabilities to classes:

Choose a threshold (usually 0.5). If predicted probability ≥ threshold → predict class 1. Otherwise → class 0.

Confusion Matrix Layout:

Predicted FALSEPredicted TRUE
Actual 0TN (True Negative)FP (False Positive)
Actual 1FN (False Negative)TP (True Positive)
Reading confusion matrices in R output: The rows are the actual values, columns are the predicted values. But watch out — exam tables may swap this! Always check the labels carefully.

7. Sensitivity & Specificity

Sensitivity (True Positive Rate) P(Test + | Condition +)

= TP / (TP + FN)

"Of all actually positive cases, how many did we correctly predict as positive?"
Specificity (True Negative Rate) P(Test − | Condition −)

= TN / (TN + FP)

"Of all actually negative cases, how many did we correctly predict as negative?"
The threshold trade-off:
• Higher threshold (e.g., 0.8): harder to classify as positive → higher specificity, lower sensitivity
• Lower threshold (e.g., 0.2): easier to classify as positive → higher sensitivity, lower specificity

Think of it like a strict vs. lenient judge.
Worked example:
Predicted FALSEPredicted TRUE
Actual 08515
Actual 11040
Sensitivity = TP / (TP + FN) = 40 / (40 + 10) = 40/50 = 0.80
Specificity = TN / (TN + FP) = 85 / (85 + 15) = 85/100 = 0.85

8. ROC Curve, AUC & Youden's J Index

The ROC curve (Receiver Operating Characteristic) shows how sensitivity and specificity change across all possible thresholds.

ROC Curve Axes
x-axis: 1 − Specificity (False Positive Rate)
y-axis: Sensitivity (True Positive Rate)
AUC = Area Under the ROC Curve
• AUC = 0.5 → random guessing (the diagonal line)
• AUC = 1.0 → perfect classification
• AUC < 0.5 → worse than random (something is wrong!)

Rule of thumb:
0.5–0.6 = poor  |  0.6–0.7 = fair  |  0.7–0.8 = good  |  0.8–0.9 = very good  |  0.9+ = excellent
Comparing models: The model with the larger AUC has better predictive accuracy. This is the standard way to compare logistic regression models in the exam.

Youden's J Index — Finding the Optimal Threshold

Youden's J Index
J = Sensitivity + Specificity − 1

The threshold that maximizes J gives the optimal balance between sensitivity and specificity. This is often asked when the exam gives you a table of thresholds.

Worked example — Threshold selection with Youden's J (2024 Exam style):
ThresholdSensitivitySpecificityJ = Sens + Spec − 1
0.200.950.500.45
0.300.900.650.55
0.400.850.780.63
0.500.750.850.60
0.600.600.900.50
0.700.400.950.35

The maximum J = 0.63 occurs at threshold = 0.40.
Therefore, the optimal classification threshold is 0.40.

Exam trap: Don't assume 0.5 is always the best threshold! The optimal threshold depends on the data and is found via Youden's J or domain-specific considerations (e.g., in medical screening, you may prefer higher sensitivity even at the cost of specificity).

9. Cross-Validation for Model Comparison

Cross-validation (CV) estimates how well a model generalizes to unseen data. The most common variant is k-fold cross-validation (typically k = 10).

How 10-fold CV works:

  1. Split the data into 10 equally-sized folds.
  2. For each fold i = 1, ..., 10:
    • Train the model on all data except fold i.
    • Test the model on fold i and record the metric (accuracy, AUC, etc.).
  3. Average the 10 metric values to get the CV estimate.
Why cross-validation?
• A single train/test split can give misleading results depending on which observations land in which set.
• CV uses every observation for both training and testing (just not at the same time).
• The CV estimate has lower variance than a single train/test split.
• Directly connected to the bias-variance tradeoff: CV helps us pick the model with the best test error, not just training error.

Comparing logistic regression models with CV:

2024 Exam — 10-fold CV comparison:

Two logistic regression models are compared using 10-fold cross-validation:

ModelCV AccuracyCV AUC (roc_auc)
fit_folds_2p (2 predictors)0.9110.830
fit_folds_6p (6 predictors)0.9130.837

Which model is better?
The 6-predictor model has both higher accuracy (0.913 > 0.911) and higher AUC (0.837 > 0.830). Therefore, the 6-predictor model has better predictive performance.

Note: The difference is small. If accuracy and AUC disagreed (one model better on accuracy, the other on AUC), you would typically prefer the model with higher AUC because AUC evaluates performance across all thresholds, while accuracy depends on a single threshold choice.

CV and the bias-variance tradeoff:
• Too few folds (e.g., 2-fold): Each training set is small → high bias in the CV estimate
• Too many folds (e.g., LOOCV = n-fold): Training sets overlap heavily → high variance in the CV estimate
• 10-fold is the standard compromise between bias and variance of the CV estimate itself

10. AIC Backward Selection for Logistic

Same principle as linear regression: start with the full model, remove variables one at a time, keep the model with the lowest AIC.

Key differences from linear regression:
• We use AIC, NOT adjusted R² (logistic regression has no R²)
• The <none> row in the table means "remove nothing" (= keep the current model)
• If <none> has the lowest AIC → stop, the current model is the best
• Otherwise, remove the variable whose removal gives the lowest AIC
Exam trap: permuted rows!
The exam often shuffles the rows in the AIC table so they're NOT in order. Don't just pick the first or last row — scan ALL rows and find the one with the smallest AIC value.
Example AIC output (rows scrambled):
RemoveAIC
x31328.15
<none>1329.96
x11331.42
x21345.80
Current AIC = 1329.96 (<none> row).
Removing x3 gives AIC = 1328.15 < 1329.96.
→ Remove x3 (it gives the lowest AIC).

Exercise 2 — Bias-Variance Tradeoff
3 pts • Course Exercises
a) State the bias-variance decomposition of the expected test MSE. Define each term. b) Consider fitting the true function f(x) = sin(x). Compare a linear regression model (y = b0 + b1*x) with KNN using k = 3. Which method would you expect to have higher bias? Which would have higher variance? Explain. c) In what situation would a non-parametric method like KNN outperform a parametric method like logistic regression?
a) Decomposition
b) Linear regression vs. KNN for sin(x)
c) When does non-parametric win?
a) E[(Y − f̂(x))²] = Var(f̂(x)) + [Bias(f̂(x))]² + Var(ε)

• Bias = E[f̂(x)] − f(x): the systematic error from approximating a real-world problem by a simpler model. How far off the model is on average from the truth.
• Variance = Var(f̂(x)): how much the estimated function f̂ changes when trained on different data sets. A model that fits noise will vary a lot.
• Irreducible error = Var(ε): the inherent noise in the data. Cannot be reduced by any model.

b) For f(x) = sin(x):
• Linear regression has higher bias: It forces a straight line onto a sinusoidal curve. No matter how much data we have, the line will systematically miss the curves of sin(x).
• KNN (k=3) has higher variance: With only 3 neighbors, KNN is very flexible and will closely follow the training data. Different training sets will produce very different predictions.
• Linear regression has lower variance: A straight line is determined by just 2 parameters, so it doesn't change much across training sets.
• KNN (k=3) has lower bias: Since sin(x) is non-linear, KNN can capture the curvature locally.

In this case, KNN would likely outperform linear regression because the true function is highly non-linear, making the bias of linear regression a bigger problem than the variance of KNN.

c) A non-parametric method outperforms a parametric method when:
• The true relationship f(x) is highly non-linear or complex
• There are enough training observations to estimate f without overfitting
• The parametric assumption (e.g., linearity) is substantially wrong, causing high bias that cannot be overcome
