Module 6: Linear Regression

Interpret slopes, R², AIC model selection, cross-validation, collinearity — every exam
Week 1 • Day 6
Concept

1. Simple Linear Regression

We fit a straight line through data to predict a response variable y from one predictor x.

Simple Linear Regression Model
ŷ = b₀ + b₁x
  • b₀ (intercept): predicted value of y when x = 0
  • b₁ (slope): for each 1-unit increase in x, y changes by b₁ on average
Residual
Residual = y − ŷ = observed − predicted

A positive residual means the model underestimated; a negative residual means it overestimated.

Slope from correlation
b₁ = r · (sy / sx)
Reading R output: In the Coefficients table, the Estimate column gives b₀ (Intercept row) and b₁ (predictor row). The Std. Error column gives SEb. The t value column is Estimate / Std. Error.
Interpretation template: "For each additional [unit of x], we expect [y] to [increase/decrease] by [|b₁|] [units of y], on average."
Concept

2. Multiple Linear Regression

With multiple predictors, the model extends to:

Multiple Linear Regression
ŷ = b₀ + b₁x₁ + b₂x₂ + ... + bkxk
"All else being equal" interpretation:
Each slope bj tells you the effect of xj on y while holding all other predictors constant.

Template: "All else being equal, for each 1-unit increase in xj, y changes by bj on average."

Categorical Predictors (Indicator Variables)

R automatically creates indicator (dummy) variables for categorical predictors. For example, if Sex has levels Female and Male:

  • SexM = 1 if the observation is male, 0 if female
  • Female is the reference category (absorbed into the intercept)
Example with indicator variable:
ŷ = 12.5 + 3.2 · Weight − 1.8 · SexM

For females (SexM = 0): ŷ = 12.5 + 3.2 · Weight
For males (SexM = 1): ŷ = (12.5 − 1.8) + 3.2 · Weight = 10.7 + 3.2 · Weight

Interpretation of SexM coefficient: "All else being equal, males have a predicted y that is 1.8 units lower than females."

Interaction Terms

An interaction term like Bwt:SexM means the slope of Bwt differs between males and females.

Model with interaction
ŷ = b₀ + b₁·Bwt + b₂·SexM + b₃·Bwt:SexM

The interaction coefficient b₃ tells you how much steeper (or flatter) the slope is for the non-reference category.

Concept

3. R² and Adjusted R²

R² measures the proportion of variance in y explained by the model. Ranges from 0 to 1.

R² (Coefficient of Determination)
R² = 1 − Σ(yi − ŷi)² / Σ(yi − ȳ)² = 1 − SSRes / SSTot
For simple regression: R² = r² (the square of the correlation coefficient).

Problem: R² always increases when you add more predictors, even useless ones. Solution:

Adjusted R²
R²adj = 1 − [Σ(yi − ŷi)² / (n − k − 1)] / [Σ(yi − ŷi)² / (n − 1)]

Equivalently:

Adjusted R² from residual standard error
R²adj = 1 − (Residual SE)² / Var(y)
Computing R²adj from R output:
1. Find Residual standard error in the output (e.g., 15.72)
2. Compute Var(y) separately (e.g., from var(data$y))
3. R²adj = 1 − (15.72)² / Var(y)
R² Always increases with more predictors.
Use for simple regression or reporting.
R²adj Penalizes for extra predictors.
Use for comparing models with different numbers of predictors.
Concept

4. Hypothesis Testing for Coefficients

For each predictor, R tests whether the coefficient is significantly different from zero:

Hypotheses
H₀: βj = 0   (predictor has no linear relationship with y)
HA: βj ≠ 0   (predictor does have a linear relationship with y)
Test statistic
T = (bj − 0) / SEbj = Estimate / Std. Error     with df = n − k − 1

where k = number of predictors in the model.

Computing T from an incomplete summary table: Sometimes the exam gives you an R summary table with a missing t-value. You must compute it yourself: T = Estimate / Std. Error. This is a direct division — nothing more.

Example (2023 retake): For the predictor "thigh" with Estimate = 0.170 and Std. Error = 0.122:
T = 0.170 / 0.122 = 1.393

Confidence Interval for a Coefficient

Confidence Interval for βj
bj ± t*(n−k−1) · SEbj

where t*(n−k−1) is the critical value from the t-distribution with df = n − k − 1 degrees of freedom.

Worked example (from 2024 exam): 95% CI for neck coefficient
Suppose the summary output gives: Estimate(neck) = −0.432, SE = 0.145, df = 245.

The 95% critical value for df = 245 is t* ≈ 1.970 (from qt(0.975, df=245)).

95% CI = −0.432 ± 1.970 × 0.145
= −0.432 ± 0.2857
= (−0.718, −0.146)

Since the interval does NOT contain 0, we conclude that neck is a significant predictor at the 5% level.
Decision rules (all equivalent):
• If p-value < α, reject H₀ → predictor is significant
• If the CI for βj does NOT contain 0, the predictor is significant
• If |T| > t*crit, reject H₀
Exam trap: The degrees of freedom for the t-test is df = n − k − 1 (residual df), NOT n − 1. Read the residual df from the R output line: Residual standard error: XX on YY degrees of freedom. YY is your df.
Concept

5. AIC Model Selection — Backward Elimination & Forward Selection

AIC (Akaike Information Criterion) balances model fit against complexity:

AIC (simplified form for regression)
AIC = constant + n · ln(RSS / n) + 2k

Lower AIC = better model. The step() function in R performs stepwise selection automatically.

Backward Elimination

  1. Start with the full model (all predictors). Note the current AIC.
  2. Try removing each variable one at a time. For each removal, compute the new AIC.
  3. Remove the variable whose removal gives the LOWEST AIC.
  4. Repeat until removing any remaining variable would increase AIC.
  5. The <none> row shows the AIC of keeping all remaining variables.

Forward Selection

Forward selection works in the opposite direction:

  1. Start with the intercept-only model (no predictors).
  2. Try adding each available variable one at a time. For each addition, compute the new AIC.
  3. Add the variable whose addition gives the LOWEST AIC.
  4. Repeat until no addition would decrease AIC.
  5. The <none> row shows the AIC of the current model without adding anything.
Example: Forward selection AIC table (from exercise 5.7)
AddDfAIC
+ abdomen11460.2
+ chest11508.7
+ hip11518.4
<none>1572.3

Current model AIC = 1572.3 (<none> row).
Adding abdomen gives AIC = 1460.2, which is the lowest and is below 1572.3 → add abdomen.
In the next step, try adding each remaining variable to the model that already includes abdomen.
Backward Elimination Start with all predictors.
Remove the weakest one at each step.
Good when you have a reasonable full model.
Forward Selection Start with intercept only.
Add the strongest predictor at each step.
Good when you have many potential predictors.
EXAM TRAP: In the step() output, the rows are often permuted/shuffled (not in the order you might expect). You must scan ALL rows to find the one with the minimum AIC value. Do NOT just pick the first or last row!
Reading step() output (backward elimination):

− age14413.35
− height14418.22
− parity14414.80

<none> AIC = 4415.11 (keeping everything).
Removing age gives AIC = 4413.35, which is lower than 4415.11 → removing age improves the model.
4413.35 is the minimum AIC → remove age.
AIC and R²adj relationship:
Lower AIC generally corresponds to higher R²adj. A model with lower AIC will typically have a larger (or equal) adjusted R² compared to a model with higher AIC.
Concept

6. Cross-Validation

Cross-validation (CV) is a technique for estimating how well a model generalizes to new, unseen data. Unlike R² or AIC which only measure fit on the training data, CV actually tests prediction on held-out data.

How k-fold Cross-Validation Works

  1. Randomly split the data into k equally sized folds (groups).
  2. For each fold i = 1, ..., k:
    • Use fold i as the test set
    • Use the remaining k − 1 folds as the training set
    • Fit the model on training data, compute prediction error (RMSE) on test data
  3. Average the k error estimates to get the overall CV error.
k-fold CV Error
CV(k) = (1/k) Σi=1k RMSEi
RMSE (Root Mean Squared Error)
RMSE = √MSE = √[(1/ntest) Σ(yi − ŷi)²]
Using CV to compare models:
• Fit each candidate model using the same k-fold split
• Compute CV error for each model
• The model with lower CV error generalizes better

Common choices: k = 5 or k = 10. With k = 10 ("10-fold CV"), each fold contains 10% of the data.

CV for Classification Models

When comparing classification models (e.g., logistic regression), CV can use different metrics instead of RMSE:

  • Accuracy: proportion of correct predictions (higher is better)
  • ROC AUC: area under the ROC curve (higher is better)
Key insight: Cross-validation gives you an honest estimate of out-of-sample performance. A model can have high R² on training data but poor CV performance if it is overfitting. Always prefer CV error over training error when choosing between models.
Exam trap: When comparing models with CV, make sure you compare the same metric. If one model reports CV accuracy and another reports CV RMSE, you cannot directly compare them. Also note: for error metrics (RMSE), lower is better; for performance metrics (accuracy, AUC), higher is better.
Concept

7. Collinearity

Collinearity (or multicollinearity) occurs when two or more predictor variables in a multiple regression model are highly correlated with each other.

Why it matters

  • Makes individual coefficient estimates unreliable — the standard errors become inflated, so t-values shrink and p-values increase
  • Coefficients can change drastically when a correlated predictor is added or removed
  • Does NOT affect overall prediction quality — the model as a whole can still predict well

How to detect collinearity

  • Correlation matrix: Compute pairwise correlations between predictors. High correlation (|r| > 0.7–0.8) suggests collinearity.
  • Symptom: A predictor you expect to be significant (based on simple regression) becomes non-significant in the multiple regression model.
Example:
Suppose you fit: ŷ = b₀ + b₁·chest + b₂·abdomen

If chest and abdomen are highly correlated (r = 0.92), then:
• Both are individually strong predictors (significant in simple regression)
• But in the joint model, one or both may appear non-significant
• The model R² is still high — overall prediction is fine
• The individual slope estimates are just hard to separate
Bottom line: Collinearity is a problem for interpretation (which predictor gets "credit"?), not for prediction. If your goal is prediction, collinearity is less of a concern. If your goal is understanding individual effects, remove or combine collinear predictors.
Concept

8. Residual Analysis — Checking Model Assumptions

After fitting a linear model, R produces 4 diagnostic plots via plot(model). Each checks a specific assumption:

Plot 1: Residuals vs Fitted

  • Checks: Linearity and constant variance
  • Good: Random scatter around 0, no pattern (no curve, no fan shape)
  • Bad: U-shape or curve → non-linear relationship. Fan/funnel shape → non-constant variance

Plot 2: Normal Q-Q

  • Checks: Normality of residuals
  • Good: Points fall approximately on the diagonal line
  • Bad: Systematic deviations from the line (heavy tails, skewness)

Plot 3: Scale-Location (Spread-Location)

  • Checks: Homoscedasticity (constant variance)
  • Good: Horizontal red line, points evenly spread
  • Bad: Increasing/decreasing trend → variance changes with fitted values

Plot 4: Residuals vs Leverage

  • Checks: Influential observations
  • Good: All points inside Cook's distance contours (dashed lines)
  • Bad: Points beyond Cook's distance > 0.5 (or > 1) are highly influential
Quick memory trick for the exam:
1. Residuals vs Fitted → Linearity + Equal variance
2. Normal Q-Q → Normality
3. Scale-Location → Spread (homoscedasticity)
4. Residuals vs Leverage → Influential points
Cheat Sheet

9. Interaction Terms & Module Summary

Writing out the simplified model for a specific category

When a model has indicator variables and interactions, substitute the category value to simplify:

Example: Cat heart weight model
Full model: ŷ = 2.98 + 2.64·Bwt − 4.17·SexM + 1.68·Bwt:SexM

For female cats (SexM = 0):
ŷ = 2.98 + 2.64·Bwt − 4.17(0) + 1.68·Bwt(0)
ŷ = 2.98 + 2.64·Bwt

For male cats (SexM = 1):
ŷ = 2.98 + 2.64·Bwt − 4.17(1) + 1.68·Bwt(1)
ŷ = (2.98 − 4.17) + (2.64 + 1.68)·Bwt
ŷ = −1.19 + 4.32·Bwt

Module 6 Summary

  1. Simple regression: ŷ = b₀ + b₁x, slope = r · (sy/sx)
  2. Residual = observed − predicted = y − ŷ
  3. Multiple regression: "all else being equal" interpretation for each slope
  4. Indicator variables: reference category absorbed into intercept
  5. Interaction terms: different slopes for different categories
  6. R² = 1 − SSRes/SSTot; for simple regression R² = r²
  7. R²adj = 1 − (Residual SE)²/Var(y) — penalizes extra predictors
  8. T-test for coefficients: T = bj/SE, df = n − k − 1
  9. Confidence interval for slope: bj ± t* · SE; if CI excludes 0 → significant
  10. AIC backward elimination: remove variable giving lowest AIC; stop when <none> is lowest
  11. AIC forward selection: add variable giving lowest AIC; stop when <none> is lowest
  12. Cross-validation: CV(k) = (1/k) Σ RMSEi; lower CV error = better generalization
  13. Collinearity: correlated predictors inflate SE but don't hurt overall prediction
  14. 4 diagnostic plots: linearity, normality, homoscedasticity, influential points

Exercise 5 — Adjusted R² Comparison
2 pts • From 2023 retake
After backward elimination, model_step has AIC = 1409.6 while model_full has AIC = 1416.9. Is the adjusted R² of model_step smaller or larger than that of model_full? Justify your answer using the relationship between AIC and adjusted R².
R²adj of model_step is larger than that of model_full.

Justification:
Both AIC and R²adj penalize for the number of parameters in the model.

• AIC = constant + n·ln(RSS/n) + 2k — lower is better
• R²adj = 1 − [RSS/(n−k−1)] / [SSTot/(n−1)] — higher is better

When a variable is removed and AIC decreases, it means the penalty reduction (from fewer parameters) outweighs the loss in fit. The same logic applies to R²adj: removing a weak predictor increases R²adj because the penalty term shrinks more than the fit worsens.

Since model_step has lower AIC (1409.6 < 1416.9), it also has higher R²adj.

Exercise 8 — Cross-Validation Model Comparison
3 pts • From 2024 exam
You are comparing two logistic regression models for predicting whether a patient has a disease. You perform 10-fold cross-validation on both models and obtain the following results: Model A (3 predictors: age, blood_pressure, cholesterol): - Mean CV Accuracy: 0.823 - Mean CV ROC AUC: 0.871 Model B (6 predictors: age, blood_pressure, cholesterol, bmi, smoking, exercise): - Mean CV Accuracy: 0.817 - Mean CV ROC AUC: 0.864 a) Explain what "10-fold cross-validation" means. What does each fold represent? b) Which model would you choose? Justify using both metrics. c) Model B has more predictors than Model A. Why might adding more predictors actually lead to worse CV performance?
a) Explain 10-fold CV
b) Model choice
c) Why more predictors can hurt
a)
In 10-fold CV, the dataset is randomly split into 10 equally sized subsets (folds). The model is trained and tested 10 times: each time, one fold is held out as the test set and the remaining 9 folds are used as the training set. The performance metric (accuracy, AUC, etc.) is computed on each test fold, and the 10 results are averaged to produce the final CV estimate.

Each fold represents approximately 10% of the data, and every observation appears in the test set exactly once.

b)
Choose Model A.

Model A has higher CV accuracy (0.823 > 0.817) AND higher CV ROC AUC (0.871 > 0.864). Both metrics indicate that Model A generalizes better to unseen data.

Additionally, Model A is simpler (3 predictors vs. 6), so by the principle of parsimony, we prefer the simpler model when performance is equal or better.

c)
Adding more predictors can lead to overfitting: the model fits noise in the training data rather than the true underlying pattern. While the training performance may improve (R² always increases), the model's ability to predict on new data can decrease. Cross-validation detects this because it evaluates on held-out data.

The additional predictors (bmi, smoking, exercise) may not carry enough predictive signal to justify the added complexity, or they may be collinear with the existing predictors, adding noise without improving predictions.
